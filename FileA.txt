Unit Testing

What is Unit Testing?
Unit testing is the execution of a complete class, routine or small program that has been written by a programmer or a team of programmers, which is testes in isolation from the more complete system.

Unit Testing is conducted by the developer using development tools in IDE.

It is a type of whitebox testing.


Integration Testing

What is Integration Testing?
Integration Testing is the combined execution of two or more classes, packages, components, or subsystems that have been created by multiple programmers or programming teams.

Integration Testing is conducted by developers.

It is a type of blackbox testing.

System Testing

System testing is the execution of the software in its final configuration, including integration with other software and hardware systems.

System testing is conducted by testers

It is type of blackbox testing


Acceptance Testing

The acceptance testing plan is created during the design phase and the actual testing is done after the system is fully developed and tested.

Acceptance testing is conducted by end users

It is type of blackbox testing
